## 第一章：X86架构和预期操作系统的特点

### 认识X86架构
> 名词x86是一种平台体系的代表，向后兼容Intel 8086 CPU的指令集。

1981年，IBM PC采用x86的指令集，x86体系已经成为通用的标准。大量软件，如操作系统(DOS, Windows, Linux, BSD, Solaris, Mac OS X)
均基于x86平台实现功能。

本书不打算面向x86 64位架构，而是x86 32位体系设计操作系统。由于向后的兼容性，我们的系统也适用于目前主流的PC(如果你想在真机上试试也许没那么容易)。

### 设计一个怎样的操作系统
把目标定为设计一个简单的类Unix系统，语言选用C++。但是，我们不打算仅仅是验证概念，预期的操作系统将具有引导、用户shell以及扩展API的功能。

总之，系统是针对x86平台，运行32位程序，并与IBM PC兼容。

设计要求：
- 基于C++实现
- x86 32位架构
- 支持Grub引导
- 具有驱动模块
- 类UNIX
- 多任务
- ELF格式的用户执行程序
- 模块化(使用/dev/...访问)
...* IDE硬盘
...* DOS分区
...* 时钟
...* 只读EXT2文件系统
...* *Boch VBE*
- 用户接口
...* Posox API
...* LibC
...* 运行shell并支持运行如Lua的程序
